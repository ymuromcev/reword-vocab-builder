# Backlog conventions

Здесь лежит «что делаем потом» для проекта `reword-vocab-builder`.
Один таск = один файл `BL-NN.md`. Этот каталог — единственное
исключение из `private/` в `.gitignore`: backlog коммитим, остальное
private — нет.

Префикс `_README` — чтобы файл сортировался наверх и пропадал из
Dataview / Bases / любых запросов вида «все BL-задачи в папке»
(фильтрация по префиксу `BL-`).

---

## Имя файла

Стандарт: `BL-NN.md`. Нумерация сквозная, заголовок живёт во
frontmatter, не в имени.

---

## Frontmatter

Обязательные поля:

```yaml
---
id: BL-NN
title: "..."
status: open | in_progress | done | archived
priority: P0 | P1 | P2 | P3
tier: XS | M | L
created: YYYY-MM-DD
tags: [...]
---
```

Опциональные:

- `closed: YYYY-MM-DD` — обязательно при `status: done` или
  `status: archived`.
- `refs: [BL-X, BL-Y]` — связанные / blocking / follow-up таски.

---

## Статусы (только эти 4)

| status | когда | что значит |
| --- | --- | --- |
| `open` | заведён, ждёт обсуждения / согласования образа / старта | можно перейти в `in_progress` или `archived` |
| `in_progress` | активно делается | можно перейти в `done` или вернуться в `open` |
| `done` | цель достигнута, закрыто | финальное; `closed:` обязателен |
| `archived` | отменён, заменён, устарел | финальное; `closed:` обязателен; в Notes — почему |

Других значений не используем.

---

## Приоритеты

- **P0** — нужно для MVP (без этого CLI не запустится).
- **P1** — нужно для полноценного использования.
- **P2** — полезное, но не блокирующее.
- **P3** — потом / nice-to-have.

## Тиры (объём работы)

- **XS** — меньше 20 строк, тривиальный фикс / стаб.
- **M** — фича 2+ файла, тесты, доку обновить.
- **L** — архитектурное / многокомпонентное / нужно RFC.

---

## Структура тела

Рекомендованные секции:

- `## Context` — почему задача появилась.
- `## Образ результата` — что пользователь увидит / получит /
  сможет сделать (user-level, не тех. язык).
- `## Plan` — что сделать (шаги).
- `## Definition of Done` — критерии завершения.
- `## NOT in scope` — что **не** входит, чтобы не разрасталось.
- `## Notes` / `## Open questions` — по необходимости.
- `## Progress` — ведём по ходу: что сделано, коммиты, факт push'а.

---

## Правила работы

1. **До кода**: формулируем user-level образ результата →
   согласовываем с пользователем → потом код. Любой нетривиальный
   таск = образ результата записан в `## Образ результата` ИЛИ в чате
   с явным «ОК».

2. **В диалоге**: user-level стиль. Тех. детали — только когда без
   них пользователь не может проверить или повторить.

3. **На каждом шаге**: `## Progress` обновляется сразу — что сделано,
   в каком коммите, факт push'а.

4. **При закрытии**:
   - `status: done` (или `archived` с причиной).
   - `closed: YYYY-MM-DD`.
   - В `## Plan` пройтись по буллетам, отметить ✅.
   - Всё, что всплыло во время работы и не вошло в DOD — выносим в
     новый BL и ссылаемся через `refs:`.

5. **Никаких неявных шагов в плане**. Всё, что делаем, должно быть в
   `## Plan` или `## Progress` конкретного BL.

---

## Поиск

- Активные: `grep -lE '^status: (open|in_progress)$' BL-*.md`.
- Закрытые: `grep -lE '^status: (done|archived)$' BL-*.md`.
- По приоритету / тиру — через `grep` по соответствующему полю.
