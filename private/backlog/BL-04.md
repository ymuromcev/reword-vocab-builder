## Context

В Reword глаголы хранятся с префиксом `to ` (`to leverage`, `to align`,
`to circle back`). Иначе они некорректно произносятся и не матчатся с
существующим словарём юзера. Нужно автоматически определять глаголы и
ставить префикс.

## Образ результата

Чистая функция:

```python
is_verb("leverage")          # True
is_verb("trade-off")         # False
is_verb("circle back")       # True (phrasal verb)
is_verb("running")           # False (gerund, не base form)

to_prefix("leverage")        # "to leverage"
to_prefix("trade-off")       # "trade-off"
to_prefix("to align")        # "to align" (уже с префиксом)
```

## Plan

- [ ] `src/verb_detector.py`: определить базовую форму глагола.
- [ ] Подход: spaCy POS-tagging (`VB` = base form), fallback на
      WordNet (`wn.VERB`).
- [ ] Edge cases:
      - phrasal verbs (`push back`, `dive into`) — детектить через
        парные паттерны.
      - идиомы (`raise the bar`, `bake in`) — да, глаголы.
      - gerunds (`running`, `building`) — НЕ префиксить.
      - participles (`built`, `running`) — НЕ префиксить.
- [ ] `to_prefix(word)` — идемпотентный (повторный вызов не дублирует
      `to to`).
- [ ] Tests на cherry-picked сете из `~/Desktop/pm_interview_vocab.csv`
      (там 79 глаголов уже размечены).

## Definition of Done

- На сете из 200+ слов precision/recall глагольной детекции > 95%.
- Phrasal verbs детектятся.
- Идемпотентность гарантирована.
- Тесты покрывают edge cases.

## NOT in scope

- Префикс для существительных (`a/an/the`) — Reword не требует.
- Грамматические правила для других частей речи.

## Notes

Подсказка по архитектуре: можно сделать конфигурируемый "manual override"
файл (`config/verb_overrides.yaml`) — список слов, которые модель
неправильно классифицирует. Полезно если spaCy ошибается на новых
идиомах.
