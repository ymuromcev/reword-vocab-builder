## Context

Сгенерированный список слов нужно отфильтровать: убрать всё, что юзер
уже знает достаточно хорошо. Иначе тул генерит "мусорный" CSV с
повторяющимися словами.

## Образ результата

```python
filtered = dedup(words, backup_index)
# → список слов, которых нет в backup'е ИЛИ они в слабом статусе
```

Правила:

- Слово в статусе `mastered` (I_REP >= 60 дней) → **skip**.
- Слово в статусе `active-long` (I_REP >= 14 дней) → **skip**.
- Все остальные статусы → **keep**.

Матчинг по lowercase + strip + удаление префикса `to `.

## Progress

Реализован чистый `src/dedup.py` по RFC-006: `dedup` возвращает
`(kept, DedupReport)`, есть тонкая обёртка `dedup_only`, `DedupReport`
заморожен и умеет рендериться в многострочную CLI-сводку с разделением
на kept / skipped блоки. Тестовый набор покрывает все 7 статусов SRS,
оба направления нормализации `to `, in-list duplicate, пустые слова и
консистентность `reasons` со списком `decisions`.
