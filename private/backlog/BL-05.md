## Context

Reword принимает на импорт только специфический формат CSV. Любое
отклонение (header, single-quote, comma-separated) → парсер падает или
теряет данные.

## Образ результата

```python
rows = [
    {"word": "to leverage", "ipa": "[ˈlevərɪdʒ]",
     "ru": "использовать", "ex1_en": "...", "ex1_ru": "...",
     "ex2_en": "...", "ex2_ru": "..."},
    ...
]
write_csv(rows, "output/pm-vocab-2026-05-19.csv")
```

Результат: файл точно в формате Reword, проверяется на эталоне
`~/Desktop/pm_interview_vocab.csv`.

## Plan

- [ ] `src/csv_writer.py`: `write_csv(rows, path)`.
- [ ] Параметры: `delimiter=";"`, `quotechar='"'`,
      `quoting=csv.QUOTE_ALL`, `lineterminator="\n"`, `encoding="utf-8"`,
      без header'а.
- [ ] Валидация на входе: все 7 ключей присутствуют, ни один не пуст.
- [ ] Тест: round-trip (записал — прочитал — сравнил).
- [ ] Тест: побайтовое сравнение output с known-good fixture.

## Definition of Done

- Output идентичен формату эталонного `pm_interview_vocab.csv`.
- Тесты проходят.
- Любая пустая ячейка → exception (не silent skip).

## NOT in scope

- Импорт CSV / парсинг — нужен только writer.
