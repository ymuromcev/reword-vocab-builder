## Context

Самая хрупкая часть — IPA-транскрипция. LLM на коротких словах часто
выдаёт неправильную US-транскрипцию (это известная проблема). Поэтому
сначала ищем в авторитетном словаре, и только если нет — fallback на
LLM с явным флагом "проверить вручную".

## Образ результата

```python
ipa = transcribe("leverage")  # "[ˈlevərɪdʒ]"
ipa = transcribe("punt")      # "[pʌnt]"
ipa = transcribe("doublelick") # None — не нашли, флаг flagged_for_review=True
```

Output на сборку CSV: если flagged_for_review — пишем `[? to verify]`
в IPA-колонку (юзер видит проблему сразу при импорте, может править
вручную). Если пусто после fallback — слово вообще не идёт в CSV.

## Plan

- [x] `src/ipa.py` (новый файл): `transcribe(word) -> IpaResult | None`,
       где `IpaResult` несёт `ipa`, `source` ("cmu"|"llm"), `needs_review`.
- [x] Pipeline:
      1. CMU Pronouncing Dictionary (через `cmudict` пакет, лежит
         офлайн). Многозначность: берём первую транскрипцию (само-
         документировано в `_lookup_cmu`); омографы — отдельный таск.
      2. LLM fallback с инжектируемым клиентом (`LLMClient` =
         `Callable[[str], str]`). На LLM-пути `needs_review=True`
         всегда — даже на видимо валидной строке.
      3. Манифест flagged words — будем собирать на стороне сборщика
         CSV (вне scope BL-09: всё, что нужно знать, — это флаг
         `needs_review` в `IpaResult`).
- [x] ARPABET → IPA: явная таблица в коде, 24 согласных + 15 гласных
       = все 39 фонем CMU. Stress 1 / 2 → `ˈ` / `ˌ`, ставится перед
       гласной. AH0 → schwa (ə), ER0 → r-coloured schwa (ɚ).
- [x] Тесты:
      - direct ARPABET→IPA (leverage, align, ship, engineer,
        vision/this/think, AH stressed vs 0, ER stressed vs 0,
        unknown-phone surfaces);
      - CMU path (leverage, align, ship, case-insensitivity,
        whitespace, multi-pronunciation `the`);
      - LLM path (mocked, всегда `needs_review=True`, не вызывается
        если CMU дал hit, корректно отрабатывает empty input и
        empty LLM response);
      - guard на полноту таблицы (39 фонем, явный set).

## Notes

CMU dict через PyPi: `pip install cmudict`. Лежит локально, не нужен
интернет. Это решающее преимущество — детерминированная транскрипция
не зависит от настроения модели.

ARPABET→IPA: единственная фрагильная точка. Решения:
- Таблица — явный `dict`, не выводится из чего-либо.
- `_UNSTRESSED_OVERRIDES` — отдельным словарём, только для AH и ER
  (документировано рядом).
- Неизвестные фонемы возвращаются как `?TOKEN?` — bug surfaces в
  CSV review, не молча портит данные.
- Stress mark всегда перед гласной (IPA convention), а не после
  CMU-цифры.

Multi-pronunciation: всегда берём `pronunciations[0]`. Это
конвенция CMU — первая запись наиболее частая. Trade-off (`read`
present/past) задокументирован в `_lookup_cmu` и явно выведен в
NOT in scope.
