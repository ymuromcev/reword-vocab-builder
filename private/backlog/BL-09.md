## Context

Самая хрупкая часть — IPA-транскрипция. LLM на коротких словах часто
выдаёт неправильную US-транскрипцию (это известная проблема). Поэтому
сначала ищем в авторитетном словаре, и только если нет — fallback на
LLM с явным флагом "проверить вручную".

## Образ результата

```python
ipa = transcribe("leverage")  # "[ˈlevərɪdʒ]"
ipa = transcribe("punt")      # "[pʌnt]"
ipa = transcribe("doublelick") # None — не нашли, флаг flagged_for_review=True
```

Output на сборку CSV: если flagged_for_review — пишем `[? to verify]`
в IPA-колонку (юзер видит проблему сразу при импорте, может править
вручную). Если пусто после fallback — слово вообще не идёт в CSV.

## Plan

- [ ] `src/ipa.py` (новый файл): `transcribe(word) -> tuple[str, bool]`.
- [ ] Pipeline:
      1. CMU Pronouncing Dictionary (через `cmudict` пакет, лежит
         офлайн).
      2. Если phrasal verb / идиома — транскрипция каждого слова
         склейкой.
      3. LLM fallback с явным промптом "US accent, IPA in square
         brackets".
      4. Манифест flagged words → output/<run>-flagged.txt для
         post-review.
- [ ] Тесты:
      - 100 слов из `pm_interview_vocab.csv` → сравнение с эталонными
        транскрипциями.
      - Idiom / phrasal verb cases.

## Definition of Done

- ≥90% слов получают корректную транскрипцию без LLM-fallback.
- Flagged words видны в отдельном файле и в CSV помечены.
- Тесты на 100-словном сете.

## NOT in scope

- UK accent (P3).
- Транскрипция омографов (`read` past vs present) — P3.

## Notes

CMU dict через PyPi: `pip install cmudict`. Лежит локально, не нужен
интернет. Это решающее преимущество — детерминированная транскрипция
не зависит от настроения модели.
