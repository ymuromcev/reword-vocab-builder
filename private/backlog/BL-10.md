## Context

Для каждого слова нужно: ru_translation + 2 английских примера + их
русские переводы. Это 5 из 7 колонок CSV. Качество промпта = качество
тула.

## Образ результата

```python
out = enrich({"word": "to leverage"})
# {
#   "word": "to leverage",
#   "ru": "использовать (по максимуму)",
#   "ex1_en": "We need to leverage existing customer data here.",
#   "ex1_ru": "Нужно по максимуму использовать существующие данные о клиентах.",
#   "ex2_en": "Let's leverage our existing partnerships first.",
#   "ex2_ru": "Давай сначала используем наши существующие партнёрства.",
# }
```

## Plan

- [ ] `src/enricher.py` (новый): `enrich(word_obj) -> dict`.
- [ ] Промпт-конструктор:
      - role: native English + native Russian translator.
      - constraints:
        * 2 примера разной длины (короткий + средний).
        * примеры показывают разный context use слова.
        * RU перевод естественный, не дословный.
        * без символов ($, %, →) в английских примерах
          (для проговаривания вслух — см. global rule).
- [ ] Source-mode bonus: если есть контекст из источника (BL-08),
      второй пример адаптируем из реальной цитаты.
- [ ] Тесты:
      - cherry-pick 10 слов → проверка валидности output'а.
      - Anti-pattern check: ни одного символа из stop-list в examples.

## Definition of Done

- Все 5 полей заполнены валидно.
- Запрещённые символы (`$`, `%`, `→`, `+`, `/`, `K`, `x`)
  отсутствуют в английских примерах.
- Тесты проходят.

## NOT in scope

- Локализация на другие языки кроме русского (P3).
- ML-генерация (только LLM сейчас).
