## Context

Первый из двух режимов: юзер вводит тему — тул выдаёт релевантный
список слов. LLM-driven: качество промпта решает всё.

## Образ результата

```bash
reword-vocab topic "PM interview vocabulary"
```

Output: ~150-250 слов, релевантных теме, без повторений и без воды.
Возвращается как список объектов с полем `word`, остальные поля
(IPA / examples / translation) дозаполняются BL-09 и BL-10.

## Plan

- [ ] `src/generators/topic.py`: `generate(topic: str, target_count=200) -> list[dict]`.
- [ ] Дизайн промпта:
      - role: эксперт по domain (детектится из темы).
      - constraints: only base forms for verbs (BL-04 потом ставит `to`).
      - format: JSON list of `{word, part_of_speech, context_note}`.
      - анти-вода: no obvious words ("project", "team", "work").
- [ ] Backend: пока claude API (через ENV токен или MCP).
- [ ] Тесты на mock'е API: проверка формы output'а, отсутствие пустых
      слов, dedup внутри батча.

## Definition of Done

- `generate("PM interview")` возвращает разумный список без повторов.
- Среди верхних 50 слов >80% совпадает с эталонным
  `pm_interview_vocab.csv` (валидация по аутентичности).
- Тест проходит на mock'е.

## NOT in scope

- Source-mode (BL-08).
- IPA / examples / translation (BL-09, BL-10).
- UI для review слов до записи в CSV (P3).
