## Context

Второй режим: юзер даёт файл (книга, статья, транскрипт) + одну
инструкцию — тул вытаскивает оттуда лексику. Сложнее topic-mode
потому что входы разнородные.

## Образ результата

```bash
reword-vocab source ~/Documents/inspired-by-cagan.pdf \
  --instruction "product management vocabulary I should learn"
```

Output: вокаб, извлечённый именно из этого источника. С контекстом из
книги (например, второй пример в CSV — это адаптированная цитата из
текста).

## Plan

- [ ] `src/generators/source.py`:
      `extract(source_path, instruction) -> list[dict]`.
- [ ] Парсеры:
      - PDF: `pypdf` или `pdfplumber`.
      - EPUB: `ebooklib`.
      - txt/md: stdlib.
      - HTML/web: `beautifulsoup4` (для скачанных статей).
- [ ] Chunking: большие файлы → batch'и по ~5K токенов.
- [ ] LLM-промпт: extract vocabulary per chunk → merge → dedup
      внутри батча.
- [ ] Сохранение контекста: для каждого слова — оригинальная фраза из
      источника (для BL-10 examples).

## Definition of Done

- 3 файла-эталона тестируем (один PDF, один EPUB, один txt).
- На каждом получаем разумный вокаб.
- Тест на чанкинг (большой файл не падает по таймауту / памяти).

## NOT in scope

- OCR для скан-PDF (P3).
- Аудио-источники (если хочется — отдельный BL под Whisper-транскрипт).
- DOCX (можно добавить если будет нужно).
