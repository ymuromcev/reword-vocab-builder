## Context

Второй режим: юзер даёт файл (книга, статья, транскрипт) + одну
инструкцию — тул вытаскивает оттуда лексику. Сложнее topic-mode
потому что входы разнородные.

## Образ результата

```bash
reword-vocab source ~/Documents/inspired-by-cagan.pdf \
  --instruction "product management vocabulary I should learn"
```

Output: вокаб, извлечённый именно из этого источника. С контекстом из
книги (например, второй пример в CSV — это адаптированная цитата из
текста).

## Plan

- [x] `src/generators/source.py`:
      `extract(source_path, instruction, *, llm) -> list[dict]`.
- [x] Парсеры:
      - PDF: `pypdf`.
      - EPUB: `ebooklib`.
      - txt/md: stdlib.
      - HTML/web: `beautifulsoup4`.
- [x] Chunking: ≤4000 токенов на чанк (heuristic `len(text)//4`).
- [x] LLM-промпт: extract vocabulary per chunk → merge → dedup по
      `normalize_key(word)`.
- [x] Сохранение контекста: каждое слово несёт `source_quote`
      (verbatim из чанка) + `context_note` с индексом чанка.
- [x] Hallucination filter: дропаем item, если `source_quote` не
      substring чанка.
