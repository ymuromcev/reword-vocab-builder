## Goal

Glue every layer-1/2 module into one shippable command. The CLI is the
canonical surface (the Claude skill in BL-12 will wrap it). After this
RFC a user can run `reword-vocab topic "..."` or
`reword-vocab source <file> --instruction "..."` and get a
Reword-importable CSV on disk.

## User-facing shape

```
reword-vocab topic "PM interview vocabulary"
→ output/2026-05-20-pm-interview-vocabulary.csv

reword-vocab source ~/Documents/inspired.pdf --instruction "PM vocabulary"
→ output/2026-05-20-inspired-pm-vocabulary.csv
```

Stdout (topic-mode, condensed):

```
→ Fetching backup via Drive MCP… (cached)
→ Backup: 1,847 words classified
→ Generating "PM interview vocabulary" (target=200)…
→ Dedup: 152 kept / 47 skipped
→ IPA: 152/152 (CMU 138 · LLM 14)
→ Enrich: 152/152
→ Verb prefix applied to 43 words
✓ output/2026-05-20-pm-interview-vocabulary.csv (152 rows)
```

If any IPA was flagged or any word failed enrich, also writes
`output/<slug>-flagged.txt` and prints `⚠ ...` line.

## CLI surface

```
reword-vocab topic <prompt> [--target-count N] [--output PATH]
                            [--backup-path PATH]
reword-vocab source <file>  --instruction <text>
                            [--output PATH] [--backup-path PATH]
```

| Flag             | Default        | Notes                                |
|------------------|----------------|--------------------------------------|
| `--target-count` | 200            | Only `topic`                         |
| `--output`       | auto-slug      | Overwrites without prompt            |
| `--backup-path`  | Drive MCP      | Local `.backup` file, skips fetch    |

Slug = `YYYY-MM-DD-<lowercased-prompt-or-filename-with-dashes>.csv`.
Non-ASCII chars dropped. Truncated at 64 chars.

## Pipeline

```
1. backup   = drive_mcp.fetch_latest_backup()  OR  --backup-path
   index    = backup_reader.read_backup(backup)
2. words    = topic.generate(prompt, target_count, llm=client)
              OR  source.extract(file, instruction, llm=client)
3. filtered, report = dedup.dedup(words, index)
   if not filtered: print summary, exit 0, no CSV
4. for each word:
       ipa, flagged = ipa.transcribe(word["word"], llm=client)
       word["ipa"] = ipa
       if flagged: collect for flagged.txt
5. enriched = enricher.enrich_many(filtered, llm=client)
   (drops failures into flagged.txt collection)
6. for each enriched word:
       word["word"] = verb_detector.to_prefix(word["word"], pos=word["part_of_speech"])
7. csv_writer.write_csv(rows, output_path)
8. print summary; write flagged.txt if any
```

## LLM client wiring

One Anthropic client constructed in `cli.main()` and injected into
`topic.generate` / `source.extract` / `ipa.transcribe` /
`enricher.enrich_many`. Module-level helper `_build_llm_client()` uses
the Anthropic SDK with `claude-sonnet-4-6` and reads
`ANTHROPIC_API_KEY` from env. If the key is missing → exit 2 with a
clear message.

## Errors → exit codes

| Condition                                 | Exit | Message                                    |
|-------------------------------------------|------|--------------------------------------------|
| Drive MCP unavailable & no `--backup-path`| 2    | "cannot reach Drive backup..."             |
| `--backup-path` doesn't exist             | 2    | "backup file not found: ..."               |
| `ANTHROPIC_API_KEY` not set               | 2    | "ANTHROPIC_API_KEY required"               |
| `UnsupportedSourceError`                  | 2    | message re-raised                          |
| `EmptySourceError`                        | 2    | "source produced no words"                 |
| Dedup leaves 0 words                      | 0    | "all words already mastered"               |
| Uncaught exception                        | 1    | traceback                                  |

## Tests (`tests/test_cli.py`)

- `test_parser_topic_minimal` — argparse parses `topic <prompt>`
- `test_parser_source_requires_instruction` — argparse error without `--instruction`
- `test_parser_target_count_default` — defaults to 200
- `test_topic_smoke_writes_csv` — monkeypatch every external module
  (`drive_mcp.fetch_latest_backup`, `backup_reader.read_backup`,
  `topic.generate`, `ipa.transcribe`, `enricher.enrich_many`),
  invoke `main()` with `argv=["topic", "fake topic"]`, assert CSV
  produced and shape matches.
- `test_source_smoke_writes_csv` — same with `source.extract` mocked.
- `test_dedup_all_filtered_no_csv` — when every word is mastered,
  exit 0 and no CSV is written.
- `test_missing_api_key_exits_2` — patch env to clear
  `ANTHROPIC_API_KEY`, expect exit 2.
- `test_slug_generation` — slug helper unit-tested for unicode,
  whitespace, length cap.

## Out of scope

- tqdm progress bars (use plain `→` lines).
- `--quiet` / `--verbose` (P3).
- Resume after interrupt (just re-run).
- Parallelising enrich (BL-10 already batches).
- Interactive review (P3).
- Log file output (stdout only).
